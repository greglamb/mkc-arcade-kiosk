# mkc-arcade-kiosk — Implementation Spec

> Self-hosted, themed MakeCode Arcade Kiosk for personal/family use. Built on
> top of `microsoft/pxt` via git submodule. Customizations stay in `overrides/`
> and are applied at build time so the submodule remains untouched.

**Repo:** `https://github.com/greglamb/mkc-arcade-kiosk`
**Deployed URL:** `https://greglamb.github.io/mkc-arcade-kiosk/`
**License:** MIT
**Versioning:** CalVer `0.YYMM.DDBB` (per Greg's coding standards)
**Companion repo:** `greglamb/mkc-arcade-kiosk-tvos` (consumes the deployed URL)
**Addenda:** [ADDENDUM-01](mkc-arcade-kiosk-ADDENDUM-01.md) — GitHub repo references in `games.json`

---

## 1. Purpose and constraints

Self-hosted fork of Microsoft's MakeCode Arcade Kiosk that:

1. Boots clean with a game list defined in the repo (no Microsoft defaults, no
   reliance on `targetconfig.json` from Microsoft's CDN).
2. Includes a native gamepad bridge polyfill that overrides `navigator.getGamepads()`
   when running inside a host WebView (e.g., a tvOS app). Inert in regular browsers.
3. Deploys to GitHub Pages on every push to `main` via GitHub Actions.
4. Stays in sync with upstream `microsoft/pxt` through monthly Dependabot
   submodule bumps.
5. Has no telemetry calls leaving the device (the `pxt.tickEvent` global is
   stubbed locally; behavior is configurable via build-time flag).

**Non-goals:**
- App Store distribution (this is the web piece — the tvOS shell is a separate repo).
- Multi-tenant — this is for one family.
- Editing or authoring games (that's still done on `arcade.makecode.com`).

## 2. Architecture overview

```
GitHub Pages serves a static CRA build →
  index.html loads (in order):
    1. pxt-stub.js          ← defines window.pxt, serves games.json
    2. native-gamepad-bridge.js  ← installs polyfill IF in native shell
    3. CRA bundle           ← Kiosk app, calls pxt.targetConfigAsync()
  →
  pxt-stub fetches games.json from same origin →
  Kiosk renders carousel from those games →
  User input arrives via:
    - real browser: navigator.getGamepads() native (Xbox/PS5 etc.)
    - native shell: polyfilled getGamepads() driven by host
    - keyboard fallback: WASD/IJKL (kiosk's built-in virtual gamepad)
```

The submodule (`vendor/pxt`) is the entire `microsoft/pxt` monorepo pinned to
a specific SHA. Only `vendor/pxt/kiosk/` and its peer `vendor/pxt/react-common/`
are actually built and shipped.

### 4.8 `scripts/apply-overrides.sh`

```bash
#!/usr/bin/env bash
# Idempotently copy our customizations into the pxt submodule's kiosk app
# so that `npm run build` produces our themed/instrumented kiosk.
#
# This script is safe to run multiple times. It does NOT modify the submodule
# in ways that would be staged for commit — submodule isolation handles that —
# but we add belt-and-suspenders .gitignore entries too.

set -euo pipefail

ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
KIOSK="$ROOT/vendor/pxt/kiosk"

if [[ ! -d "$KIOSK" ]]; then
  echo "ERROR: submodule not initialized. Run: npm run submodule:init" >&2
  exit 1
fi

echo "==> Copying public/ overrides"
cp -f "$ROOT/overrides/public/native-gamepad-bridge.js" "$KIOSK/public/"
cp -f "$ROOT/overrides/public/pxt-stub.js"              "$KIOSK/public/"

echo "==> Copying games.json -> kiosk public root"
cp -f "$ROOT/overrides/games.json" "$KIOSK/public/games.json"

echo "==> Patching package.json homepage for relative asset paths"
# Setting homepage to "." makes CRA emit relative URLs in index.html and the
# manifest, so the build works at any subpath (GitHub Pages project URL,
# custom domain, or being loaded as a file:// URL by a native shell).
node -e "
  const f = '$KIOSK/package.json';
  const p = require(f);
  p.homepage = '.';
  require('fs').writeFileSync(f, JSON.stringify(p, null, 2) + '\n');
"

echo "==> Injecting <script> tags into index.html (idempotent)"
INDEX="$KIOSK/public/index.html"
MARKER="<!-- mkc-arcade-kiosk injected -->"

if ! grep -q "$MARKER" "$INDEX"; then
  # Insert just before </head>. ORDERING MATTERS:
  #   1. pxt-stub.js defines window.pxt before any code references it.
  #   2. native-gamepad-bridge.js installs the getGamepads override before
  #      Kiosk's GamepadManager.initialize() calls it on app startup.
  sed -i.bak "s|</head>|    $MARKER\\
    <script src=\"%PUBLIC_URL%/pxt-stub.js\"></script>\\
    <script src=\"%PUBLIC_URL%/native-gamepad-bridge.js\"></script>\\
  </head>|" "$INDEX"
  rm -f "$INDEX.bak"
fi

echo "==> Done"
```

Set executable: `chmod +x scripts/apply-overrides.sh`.

### 4.9 `scripts/bump-submodule.sh`

```bash
#!/usr/bin/env bash
# Bump the pxt submodule to the latest master and commit.
# Use sparingly — prefer Dependabot's monthly PR which gives you a diff to review.

set -euo pipefail

ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
cd "$ROOT/vendor/pxt"

git fetch origin master
OLD_SHA=$(git rev-parse --short HEAD)
git checkout origin/master
NEW_SHA=$(git rev-parse --short HEAD)

  echo "Already at latest ($NEW_SHA). No bump needed."
  exit 0
fi

cd "$ROOT"
git add vendor/pxt
git commit -m "deps(pxt): bump submodule $OLD_SHA -> $NEW_SHA"

echo ""
echo "Bumped pxt: $OLD_SHA -> $NEW_SHA"
echo "Review with: cd vendor/pxt && git log --oneline $OLD_SHA..$NEW_SHA -- kiosk/ react-common/"
echo "Then push and let the workflow validate the build."

### 4.10 `overrides/public/pxt-stub.js`

```javascript
// pxt-stub.js — stub the upstream `pxt` global so mkc-arcade-kiosk runs
// standalone without the rest of the PXT framework.
//
// Responsibilities:
//   1. Provide pxt.targetConfigAsync() that returns our games from games.json.
//   2. Provide no-op telemetry (configurable for debugging).
//   3. Provide minimal Util helpers that Kiosk references.
//   4. Disable backend cloud calls (Kiosk ID flow is unused — we own the list).
//
// Load order: this MUST load before native-gamepad-bridge.js and before the
// CRA bundle, because Kiosk's App.tsx calls pxt.targetConfigAsync() on mount.
(function () {
  'use strict';

  if (window.pxt) {
    console.warn('[pxt-stub] window.pxt already exists; not overriding');
    return;
  }

  // ---- Debug toggle -----------------------------------------------------
  // Set via build-time env var MKC_DEBUG=true OR by adding ?mkcDebug=1 to URL.
  // When enabled, telemetry calls log to console and increment counters
  // accessible at window.__pxtStubStats.
  var DEBUG = (function () {
    try {
      // Build-time injection placeholder. apply-overrides.sh leaves this as
      // a literal; CRA's process.env substitution at build time fills it.
      var fromEnv = ('%MKC_DEBUG%' === 'true');
      var fromUrl = /[?&]mkcDebug=1\b/.test(location.search);
      return fromEnv || fromUrl;
    } catch (e) { return false; }
  })();

  var stats = {
    tickEventCount: 0,
    reportErrorCount: 0,
    reportExceptionCount: 0,
    eventsByName: {},
  };

  if (DEBUG) {
    window.__pxtStubStats = stats;
    console.log('[pxt-stub] DEBUG enabled — stats at window.__pxtStubStats');
  }

  function tick(category) {
    stats.tickEventCount++;
    stats.eventsByName[category] = (stats.eventsByName[category] || 0) + 1;
    if (DEBUG) console.debug('[pxt.tickEvent]', category);
  }

  // ---- targetConfigAsync ------------------------------------------------
  // Kiosk's App.tsx calls this on mount and expects { kiosk: { games: [...] } }.
  // We resolve the path relative to document.baseURI so it works on:
  //   - GitHub Pages project URL: greglamb.github.io/mkc-arcade-kiosk/
  //   - Any custom domain
  //   - file:// URL when bundled inside a native shell
  var cachedConfig = null;
  function targetConfigAsync() {
    if (cachedConfig) return Promise.resolve(cachedConfig);
    var url = new URL('games.json', document.baseURI).href;
    return fetch(url, { cache: 'no-cache' })
      .then(function (r) {
        if (!r.ok) throw new Error('games.json HTTP ' + r.status);
        return r.json();
      })
      .then(function (data) {
        var games = (data && Array.isArray(data.games)) ? data.games : [];
        if (DEBUG) console.log('[pxt-stub] loaded', games.length, 'games');
        cachedConfig = { kiosk: { games: games } };
        return cachedConfig;
      })
      .catch(function (err) {
        console.error('[pxt-stub] games.json load failed:', err);
        cachedConfig = { kiosk: { games: [] } };
        return cachedConfig;
      });
  }

  // ---- The stub ---------------------------------------------------------
  window.pxt = {
    targetConfigAsync: targetConfigAsync,
    tickEvent: function (cat) { tick(cat); },
    debug: function () { if (DEBUG) console.debug.apply(console, arguments); },
    log:   function () { if (DEBUG) console.log.apply(console, arguments); },
    reportError: function (cat, msg) {
      stats.reportErrorCount++;
      console.error('[pxt.reportError]', cat, msg);
    },
    reportException: function (e) {
      stats.reportExceptionCount++;
      console.error('[pxt.reportException]', e);
    },
    // Kiosk's BackendRequests.ts uses pxt.Cloud.apiRoot to talk to Microsoft's
    // kiosk code service. We point it at about:blank so any accidental
    // invocation fails loudly rather than leaking data.
    Cloud: { apiRoot: 'about:blank' },
    Util: {
      escapeForRegex: function (s) {
        return String(s).replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
      },
    },
  };

  if (DEBUG) console.log('[pxt-stub] installed');
})();
```

Note on the `'%MKC_DEBUG%'` literal: CRA does NOT do percent-substitution in
files under `public/`. That's intentional — we don't want CRA touching this.
Instead, the `apply-overrides.sh` script does a post-copy `sed` substitution to
replace `%MKC_DEBUG%` with the actual value of `$MKC_DEBUG` from the environment.

**Update `apply-overrides.sh`** to add this substitution after copying the stub:

```bash
# Substitute %MKC_DEBUG% in pxt-stub.js with the env var (or 'false')
DEBUG_VAL="${MKC_DEBUG:-false}"
sed -i.bak "s/%MKC_DEBUG%/$DEBUG_VAL/g" "$KIOSK/public/pxt-stub.js"
rm -f "$KIOSK/public/pxt-stub.js.bak"
```

Insert immediately after the `cp -f` for `pxt-stub.js`. (The final
`apply-overrides.sh` should include this substitution — implementer must merge.)

### 4.11 `overrides/public/native-gamepad-bridge.js`

```javascript
// native-gamepad-bridge.js — bridges a host WebView's native game controllers
// into the page's navigator.getGamepads() API.
//
// In a regular browser, this script no-ops and the real Gamepad API is used.
// In a native shell (detected via window.webkit.messageHandlers.gameController),
// it overrides getGamepads() and the host pushes state via __nativeGamepadUpdate.
//
// Load order: this MUST load before the CRA bundle, because Kiosk's
// GamepadManager.initialize() calls getGamepads() once at startup.
(function () {
  'use strict';

  var HANDLER_NAME = 'gameController';
  var bridge = window.webkit
    && window.webkit.messageHandlers
    && window.webkit.messageHandlers[HANDLER_NAME];

  if (!bridge) {
    // No native shell. Let the real Gamepad API work normally.
    return;
  }

  console.log('[native-gamepad-bridge] Installing (native shell detected)');

  var MAX_PADS = 4;
  var padStates = new Array(MAX_PADS).fill(null);

  /**
   * Build a W3C Standard Gamepad object from a raw state payload.
   *
   * The native side sends:
   *   {
   *     id: string,            // human-readable name
   *     buttons: number[17],   // each 0..1, standard mapping order
   *     axes:    number[4]     // each -1..+1, [leftX, leftY, rightX, rightY]
   *   }
   *
   * Standard button index reference (W3C Gamepad spec):
   *   0=A, 1=B, 2=X, 3=Y,
   *   4=L1, 5=R1, 6=L2, 7=R2,
   *   8=Select/Back, 9=Start,
   *   10=L3, 11=R3,
   *   12=DPadUp, 13=DPadDown, 14=DPadLeft, 15=DPadRight,
   *   16=Home/Guide
   */
  function build(index, raw) {
    return {
      id: raw.id || ('Native Controller ' + index + ' (STANDARD GAMEPAD)'),
      index: index,
      connected: true,
      mapping: 'standard',
      timestamp: performance.now(),
      buttons: raw.buttons.map(function (v) {
        return {
          pressed: v > 0.5,
          touched: v > 0.0,
          value: v,
        };
      }),
      axes: raw.axes.slice(0, 4),
      vibrationActuator: null,
      hapticActuators: [],
    };
  }

  /**
   * Native shell calls this on every controller state change.
   * Accepts JSON string or pre-parsed array of length MAX_PADS.
   * Fires gamepadconnected/gamepaddisconnected events on slot transitions.
   */
  window.__nativeGamepadUpdate = function (payload) {
    try {
      var incoming = typeof payload === 'string' ? JSON.parse(payload) : payload;
      for (var i = 0; i < MAX_PADS; i++) {
        var prev = padStates[i];
        var next = incoming[i] ? build(i, incoming[i]) : null;

        if (!prev && next) {
          window.dispatchEvent(new GamepadEvent('gamepadconnected', { gamepad: next }));
        } else if (prev && !next) {
          window.dispatchEvent(new GamepadEvent('gamepaddisconnected', { gamepad: prev }));
        }
        padStates[i] = next;
      }
    } catch (e) {
      console.error('[native-gamepad-bridge] update failed:', e);
    }
  };

  // Override the Gamepad API. Kiosk's GamepadManager polls this every
  // GamepadPollLoopMilli (50ms by default).
  navigator.getGamepads = function () { return padStates; };

  // Signal native that the polyfill is ready and it can push initial state.
  try {
    bridge.postMessage({ type: 'polyfill_ready' });
  } catch (e) {
    console.error('[native-gamepad-bridge] ready signal failed:', e);
  }
})();
```

# overrides/

This directory holds all customizations applied to the upstream `microsoft/pxt`
submodule at build time. Never edit files inside `vendor/pxt/` — they will be
overwritten on the next build and any changes you make there will be lost when

- `public/native-gamepad-bridge.js` — gamepad API polyfill for native shells

  "id": "12345-67890-12345-67890",
  "name": "Display name",
  "description": "One-sentence description",
  "highScoreMode": "SingleAscending"
}
```

5. Commit and push. The Pages workflow rebuilds automatically.

## Removing a game

Just remove its entry from `games.json` and push.

# mkc-arcade-kiosk

Self-hosted, themed [MakeCode Arcade Kiosk](https://arcade.makecode.com/kiosk)

## How it works

This repo is a thin wrapper around the upstream Microsoft Kiosk codebase. The
upstream is pulled in as a git submodule at `vendor/pxt`. Our customizations
live in `overrides/` and are applied at build time by `scripts/apply-overrides.sh`,
which copies our overrides into the submodule before building. The submodule
itself is never modified, which keeps `git submodule update` clean.

cd mkc-arcade-kiosk

```

Already cloned? `npm run submodule:init` to initialize submodules.

## 5. Implementation order for Claude Code

Execute in this order. Each step has acceptance criteria.

### Step 1: Create the repo skeleton

1. Create local directory `mkc-arcade-kiosk/` and `cd` in.
2. `git init -b main`
3. Create the files from §4.1 (`.gitignore`), §4.2 (`.nvmrc`), §4.3 (`LICENSE`),
   §4.4 (`package.json`), §4.14 (`README.md`), `SPEC.md` (this file).
4. Create directory structure: `mkdir -p .github/workflows overrides/public scripts vendor`
5. `git add -A && git commit -m "Initial scaffold"`

**Acceptance:** `git status` clean. `cat package.json` shows correct content.

### Step 2: Add the submodule and pin it

1. `git submodule add https://github.com/microsoft/pxt vendor/pxt`
2. Browse https://github.com/microsoft/pxt/commits/master, pick a commit SHA
   that is at least 7 days old.
3. `cd vendor/pxt && git checkout <SHA> && cd ../..`
4. `git add vendor/pxt && git commit -m "Pin pxt submodule to <SHA>"`

**Acceptance:** `.gitmodules` exists. `git submodule status` shows the pinned
SHA. `ls vendor/pxt/kiosk/src/` lists the kiosk source.

### Step 3: Add the override files

1. Create §4.10 (`overrides/public/pxt-stub.js`) — exactly as specified.
2. Create §4.11 (`overrides/public/native-gamepad-bridge.js`) — exactly as specified.
3. Create §4.12 (`overrides/games.json`) — the starter list.
4. Create §4.13 (`overrides/README.md`).
5. `git add overrides/ && git commit -m "Add override files"`

**Acceptance:** Files exist with content matching the spec byte-for-byte.

### Step 4: Add the build scripts

1. Create §4.8 (`scripts/apply-overrides.sh`).
2. **Merge the `%MKC_DEBUG%` substitution block** from the §4.10 callout into
   the script. The full script must include the `sed` substitution after the
   `cp -f` for `pxt-stub.js`.
3. Create §4.9 (`scripts/bump-submodule.sh`).
4. `chmod +x scripts/*.sh`
5. `git add scripts/ && git commit -m "Add build scripts"`

**Acceptance:** Scripts are executable. Running `./scripts/apply-overrides.sh`
locally (with submodule initialized) succeeds and produces:
- `vendor/pxt/kiosk/public/native-gamepad-bridge.js` (copy of override)
- `vendor/pxt/kiosk/public/pxt-stub.js` (copy of override, with `%MKC_DEBUG%`
  replaced by `false`)
- `vendor/pxt/kiosk/public/games.json` (copy of override)
- `vendor/pxt/kiosk/package.json` has `"homepage": "."`
- `vendor/pxt/kiosk/public/index.html` has two new `<script>` tags before `</head>`
- Running the script a second time produces no further changes (idempotent —
  verify with `git -C vendor/pxt diff --stat`)

### Step 5: Add the GitHub workflows

1. Create §4.6 (`.github/dependabot.yml`).
2. Create §4.7 (`.github/workflows/deploy.yml`).
3. `git add .github/ && git commit -m "Add CI workflows"`

**Acceptance:** YAML is valid (`yamllint` or `actionlint` clean if available).

### Step 6: Local build validation

Before pushing, validate the full build locally:

```bash
./scripts/apply-overrides.sh
cd vendor/pxt/kiosk
npm ci
CI=false npm run build
```

**Acceptance:**
- Build completes with zero errors. Warnings are acceptable.
- `vendor/pxt/kiosk/build/index.html` contains the two injected `<script>` tags.
- `vendor/pxt/kiosk/build/games.json` exists and is valid JSON matching the
  override.
- `vendor/pxt/kiosk/build/pxt-stub.js` and `native-gamepad-bridge.js` exist.

### Step 7: Push and verify deployment

1. Create the remote: `gh repo create greglamb/mkc-arcade-kiosk --public --source=. --remote=origin --push`
2. In GitHub web UI: Settings → Pages → Source: **GitHub Actions**.
3. Wait for the `Deploy to GitHub Pages` workflow to complete (~3 min).
4. Open `https://greglamb.github.io/mkc-arcade-kiosk/` in Chrome.

**Acceptance criteria:**
- Page loads with the Kiosk carousel.
- Six starter games visible.
- Browser console shows no errors (warnings about deprecated APIs from CRA are OK).
- With an Xbox/PS5 controller paired to the Mac (via Bluetooth), pressing the
  d-pad cycles through games and A launches a game.
- Pressing keyboard arrows + Space also works (kiosk's built-in fallback).
- `?mkcDebug=1` appended to the URL produces `[pxt-stub]` log lines in the console.

### Step 8: Tag a release

## 6. Test plan

Add these as Jest tests under `tests/` in the root repo. They run independently
of the kiosk build and validate the override files in isolation.

### 6.1 `tests/pxt-stub.test.js`

Tests for `overrides/public/pxt-stub.js`:

- ✅ Defines `window.pxt` if not present
- ✅ Skips installation if `window.pxt` already exists
- ✅ `pxt.tickEvent` is callable and returns nothing
- ✅ `pxt.tickEvent` increments `__pxtStubStats.tickEventCount` when DEBUG=true
- ✅ `pxt.tickEvent` does NOT touch `__pxtStubStats` when DEBUG=false
- ✅ `pxt.Cloud.apiRoot === 'about:blank'`
- ✅ `pxt.Util.escapeForRegex('a.b*c')` returns `'a\\.b\\*c'`
- ✅ `pxt.targetConfigAsync()` fetches games.json and returns `{ kiosk: { games } }`
- ✅ `pxt.targetConfigAsync()` caches result (second call doesn't refetch)
- ✅ `pxt.targetConfigAsync()` returns empty games on fetch failure

Mock `fetch` and `document.baseURI`. Use jsdom environment.

### 6.2 `tests/native-gamepad-bridge.test.js`

Tests for `overrides/public/native-gamepad-bridge.js`:

- ✅ No-ops when `window.webkit.messageHandlers.gameController` is undefined
- ✅ Installs polyfill when handler exists
- ✅ `navigator.getGamepads()` returns array of length 4
- ✅ All slots initially null
- ✅ `__nativeGamepadUpdate` with one gamepad populates slot 0
- ✅ `__nativeGamepadUpdate` slot transition from null → object dispatches `gamepadconnected`
- ✅ `__nativeGamepadUpdate` slot transition from object → null dispatches `gamepaddisconnected`
- ✅ Gamepad has `mapping === 'standard'`
- ✅ Buttons array has correct shape (`{pressed, touched, value}`)
- ✅ Button value > 0.5 → pressed=true
- ✅ `__nativeGamepadUpdate` accepts JSON string OR pre-parsed array
- ✅ Malformed payload logs error but doesn't throw

Mock `window.webkit.messageHandlers.gameController = { postMessage: jest.fn() }`.

### 6.3 `tests/games-json.test.js`

Tests for `overrides/games.json`:

- ✅ Valid JSON
- ✅ Has top-level `games` array
- ✅ Each game has `id`, `name`, `description`, `highScoreMode` (all strings)
- ✅ `highScoreMode` is one of `"SingleAscending"`, `"None"`
- ✅ `id` matches `/^_[a-zA-Z0-9]+$/` OR `/^\d{5}-\d{5}-\d{5}-\d{5}$/`

### 6.4 `tests/apply-overrides.test.js`

Integration test using `bats-core` or a Node test that:
- ✅ Creates a fake kiosk directory structure
- ✅ Runs `apply-overrides.sh` against it
- ✅ Verifies all expected files are present
- ✅ Verifies `homepage: "."` in package.json
- ✅ Verifies `<script>` tags present in index.html
- ✅ Verifies running twice produces no further changes (idempotent)

Coverage target: ≥75% per Greg's standards.

## 7. Risks and mitigations

| Risk | Mitigation |
|---|---|
| Upstream `microsoft/pxt` changes Kiosk's data flow (e.g., `App.tsx` no longer calls `pxt.targetConfigAsync`) | Dependabot PR triggers build; if build fails or smoke test breaks, don't merge. Stay on previous SHA until fix is in. |
| Upstream changes the `cfg.kiosk.games` shape | Smoke test in deploy workflow (see step 7 acceptance) would catch this. Add an explicit Jest test asserting the shape. |
| Upstream adds new `pxt.*` calls the stub doesn't cover | Console shows `pxt.xxx is not a function`. Extend stub. Failure mode is loud, not silent. |
| Polyfill conflicts with a real browser's Gamepad API | Won't happen — the polyfill only activates when `window.webkit.messageHandlers.gameController` is present, which only the tvOS shell creates. |
| `homepage: "."` breaks something in CRA's manifest paths | Validated by step 6/7 acceptance. If it ever does, fallback is to set `homepage: "https://greglamb.github.io/mkc-arcade-kiosk"` explicitly. |
| GitHub Pages rate-limits or goes down | Static hosting elsewhere (Cloudflare Pages, Netlify) is a trivial swap — workflow output is just a static dir. |

## 8. Out of scope for v1

These are deferred to v1.x:

- Custom theming (Quirktree colors, fonts)
- Per-kid profiles (Elliot's games vs Henri's games)
- Offline support (cached game PNGs)
- Telemetry to Greg's own backend (currently silent no-op)
- Analytics
- A11y improvements beyond what upstream Kiosk provides

The companion repo `mkc-arcade-kiosk-tvos` adds the native shell, controller
bridge, and app-store-friendly packaging. See its SPEC.md.
